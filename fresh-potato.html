Fresh Potatoes

Full-stack development of an efficient movie searching database solution

Time: Jan 2021

Duration: three months

Role: Client-side developer

Overview

Developed by a team of six students to provide a simplified and more efficient movie searching process for any users that watches movies and TV shows at any age range. Fresh Potato aims to solve the problem by implementing more defined and detailed recommendation options as well as filtering the movies and shows based on the streaming platforms the users are on with the help of Django web framework and TMDB (The Movie Database) API.

My role in the group was designing the website and making sure client-side view is properly displaying movies from the server-side logic.

Tech tools

Landing Page

Requirements

> Aesthetic and minimalist design

> Flexibility and efficiency of use

> Minimize the user's memory load by creating a "watch later" page

Process

I first established the layout for the landing page as well as the style of each elements on the page. I found displaying 6 movies for each section on the landing page to be the most efficient and asthetic. Then expanded the same styles to the other pages on our website to achieve consistancy. Finally, I communicated with the server-side team and fetched data to be displayed on the client-side properly.

Features

01. Movie Info

This application utilizes API from TMDb to display the information for the movies.

02. User Profile

User preferences alter the recommendation algorithm.

03. "Potato Farm"

The watch-later page which users can save the movies for later.

Reflection

The biggest challenge we encountered as a group was being unfamiliar with the technical tools for this project. Being me and my group’s first time building a full-stack project, we were lost at the beginning as it was our first time using model-view-controller framework as well as client side languages of html and css.

A big takeaway from this project for me was how to utilize online resources effectively and efficiently. Though the start was rough and our development was delayed by almost 2 weeks. I learned the basics of html and css as a client-side developer, as well as being a better team-player by communicating with the server-side team to ensure the front-end displayed data correctly.
